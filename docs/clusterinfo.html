Cluster Information

Getting information on a single cluster

Pressing the F4 key sets the system into cluster info mode. The cursor becomes a pointing finger . Left clicking on a cluster produces output in the information window similar to that shown below.

 	Cluster No. 0 - Neighbourhood Limit = 100 nm
	Area = 184316 nm2 -- Alpha = 700 -- Convex Hull Area = 289899 nm2
	# Blinks = 4112 - Density = 22310/μm2
	# Points = 4029 - Density = 21859/μm2
	No. of Subclusters = 4 - Areas: 166814, 6892, 2179, 8431 nm2 
	NND = 259.0 nm - NearestArea = 2107.5 nm2
    Min. Ellipse axes:
      Semi-Minor = 313.0 nm - Semi-Major = 390.1 nm
	Tetramers = 64 - Occupancy = 0.282 (27 nm) = 0.348 (30 nm)
	Blinks/Tetramer = 64

The last two lines concerning tetramers only appear if tetramer squares have been added to the cluster. Two values are given for the occupancy dependent on the tetramer size

While in cluster info mode the only keys that can be pressed are Esc or F4 to exit the mode.

Saving cluster information


Ticking this box and then pressing "Modify" produces a file which contains the following parameters for each cluster in the image.

  
   ClustNo NumBlinks NumPoints Area NND NearestArea SemiMinorAxis SemiMajorAxis...
     ConvexHullArea NumSubClusters(nsb) SubClusterAreas(1..nsb)

Values are separated by a space, one line per cluster. The lines have a variable length dependent on the number of subclusters. The data is saved as an ASCII text file in the same directory as the blink file. If the blink data file is named fn.3dSTM the cluster data file will be named fn_NL[NL value].clustval where NL is the current neighbourhood limit (eg. fn_NL100.clustval)
