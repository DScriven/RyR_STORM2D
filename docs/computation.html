Setting Computational Parameters

There are four values that must be set before the blink data set can be loaded (values in the image are defaults):

Neighbourhood limit

The neighbourhood limit (NL) has a value between 30 and 300 nm. The first time an analysis is done of a data set, a 1st pass at 30 nm is done to calculate the base cluster areas followed by a second pass to calculate the size of the clusters at the NL chosen. All subsequent invocations of the NL on the current data set will use the base and a 2nd pass. Cluster data is not saved between different limits, so to avoid repeat calculations it is recommended that the "Save cluster data" box be clicked.

NOTE: The cluster blink data file saved with the "Save cluster blinks" box is independent of the NL and need only be saved once per 3dSTM blink file.

Minimum Blinks per Tetramer

This value is checked first after the clusters are calculated. If this value is not met no further analysis is done and the points are added to the excluded blinks list.

Minimum Tetramer Area

This is the minimum area for a valid tetramer (default 600 nm2). As for the minimum blinks per tetramer, clusters that do not meet that criteria are added to the excluded blinks list. There are two other hidden criteria - for small clusters (< 2000 nm2) the minimum ellipse must have a minor axis (> 24 nm) so as to exclude lines of blinks that may have an area above the minimum. The second hidden criterion checks whether the blinks to points ratio is too high, so excluding fiduciary beads.

Alpha value

The alpha value (default 700) defines the way the boundaries of the base clusters are defined. Large holes will be detected and subtracted from the cluster area. Reducing the value to 400 (say) will improve the detection of small interior holes but the outer surface will get rougher. Reducing the alpha below 350 is not recommended. Increasing the alpha above 700 loosens the boundary fit.

All of these values can be changed and the output altered with the "Modify" key.

Frame numbers

The frame numbers (i.e the number of images collected) can only be changed after the first run. It is limited by the values shown after the first analysis. Changing these numbers means that the base clusters have to be recalculated, necessitating two passes. These base clusters remain true of for whatever NLs are used and until the next change in the frame range.
